This problem is fully supported by version 2021.9 of the EduTools plugin and newer. To avoid possible errors, please make sure that the plugin is updated. Alternatively, you can solve it using the web interface.

The program has one POST endpoint that receives a task as a JSON object with two fields: name (string) and description (string).

Your task is to analyze the code and make some changes so that the program would only accept the tasks that meet the following requirements:

  • The length of the name must be from 1 to 50 inclusive.
  • The length of the description must be from 1 to 200 inclusive.
  • Both fields must not be blank.

If these requirements are met, the program should respond with 200 OK status code, otherwise 400 Bad Request.

Example 1: POST /tasks request with the following body:

{
   "name": "task1",
   "description": "abc"
}

Status code: 200 OK

Example 2: POST /tasks request with the following body:

{
   "name": "",
   "description": "a"
}

Status code: 400 Bad Request
